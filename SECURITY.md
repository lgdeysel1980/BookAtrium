# Security Policy

## Reporting a Security Vulnerability

Please do not report security vulnerabilities through public GitHub Issues or Discussions.

A security vulnerability may include:

- Exposure of personal or private information
- Unauthorised access to files or application data
- Unsafe handling of credentials or authentication information
- Code execution or privilege escalation
- Installer or update tampering
- A vulnerability that could affect other BookAtrium users

For security-related reports, please contact the BookAtrium maintainers privately.

A dedicated private security reporting address or GitHub private vulnerability reporting process will be published before the first public release.

## What to Include

Please include:

- The affected BookAtrium version
- Your Windows version
- A clear description of the vulnerability
- Steps to reproduce the problem
- The possible impact
- Any suggested mitigation
- Screenshots or logs where appropriate

Do not include:

- Passwords
- Licence keys
- Private ebook files
- Personal information
- Customer data
- Database backups
- Sensitive system information unless specifically requested

## Supported Versions

BookAtrium is currently under active development.

Supported release versions will be listed here once the first public release is available.

| Version | Supported |
|---|---|
| Pre-release development builds | No public security support |
| Latest stable release | Planned |
| Older stable releases | To be confirmed |

## Disclosure

Please allow reasonable time for the issue to be investigated and corrected before publishing details publicly.
