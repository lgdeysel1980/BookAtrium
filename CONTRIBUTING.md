# Contributing to BookAtrium

Thank you for helping improve BookAtrium.

This public repository is used for:

- Bug reports
- Feature requests
- Installation and update problems
- Documentation corrections
- General user discussions
- Public releases and manuals

The main BookAtrium application source code is maintained separately and is not hosted in this repository.

## Reporting Bugs

Use the structured Bug Report form:

[Report a bug](../../issues/new?template=01-bug-report.yml)

Before submitting:

- Search existing issues
- Confirm the problem still occurs in the latest release
- Include clear reproduction steps
- Include the BookAtrium version
- Include your Windows version
- Remove all private or sensitive information

## Requesting Features

Use the Feature Request form:

[Request a feature](../../issues/new?template=02-feature-request.yml)

A useful feature request explains:

- The problem or limitation
- The proposed improvement
- How the feature would be used
- Any alternatives or workarounds

## Installation and Update Problems

Use the installation form:

[Report an installation or update problem](../../issues/new?template=03-installation-update.yml)

Please include any installer error code, the installed version, and the version you are trying to install.

## Documentation Corrections

Use the documentation form:

[Report a documentation problem](../../issues/new?template=04-documentation.yml)

Please identify the page, section, or screen as accurately as possible.

## General Questions

Use GitHub Discussions for general help and product conversations:

[Open BookAtrium Discussions](../../discussions)

## Pull Requests

Pull requests are currently limited to repository collaborators.

This repository may accept documentation corrections and public repository maintenance changes from approved collaborators.

Application source-code contributions are not currently accepted through this repository.

## Privacy and Sensitive Information

Never submit:

- Passwords
- Licence keys
- Private ebooks
- Personal documents
- Database backups
- Customer information
- Confidential business information
- Unedited logs containing sensitive data

Public issues and discussions can be viewed by anyone.

## Expected Conduct

Please remain respectful and constructive.

Reports or comments containing abuse, spam, harassment, or private information may be edited, hidden, locked, or removed.
